## üöÄ Deploy to Vercel

**Important:** This project uses React 19 and requires the `--legacy-peer-deps` flag for npm installations. This is already configured in both `vercel.json` and `.npmrc`.

### Quick Deploy

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy (will use settings from vercel.json)
vercel
```

### Option 1: Direct Deploy (No GitHub Required)

1. Install Vercel CLI:
```bash
npm i -g vercel
```

2. Deploy from your project directory:
```bash
vercel
```

3. Follow the prompts:
   - Set up and deploy? **Yes**
   - Which scope? Select your account
   - Link to existing project? **No**
   - Project name? **ma-transform-lab**
   - Directory? **./  (press Enter)**
   - Override settings? **No**

4. Your site will be live at: `https://ma-transform-lab.vercel.app`

### Option 2: Vercel Dashboard

1. Go to [vercel.com](https://vercel.com)
2. Click "Add New Project"
3. Select "Import from Git" or drag and drop your project folder
4. Configure:
   - Framework Preset: **Next.js**
   - Build Command: `npm run build`
   - Output Directory: `.next`
   - Install Command: `npm install`
5. Click "Deploy"

## üîß Troubleshooting

### Deployment Issues

If you encounter npm dependency errors during deployment:

1. **Ensure `.npmrc` file exists** with `legacy-peer-deps=true`
2. **Verify `vercel.json`** has `"installCommand": "npm install --legacy-peer-deps"`
3. **Clear Vercel cache** in your Vercel dashboard ‚Üí Settings ‚Üí Clear Build Cache
4. **Redeploy** after clearing cache

### Local Development Issues

If you get peer dependency warnings locally:

```bash
npm install --legacy-peer-deps
```
